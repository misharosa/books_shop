- використай json-server для них (тепер прийдеться робити запити)
- GoodList повинен приймати дані з пропсів
- тепер у тебе буде роутинг складатися так:
        / (головна з посиланнями на /books та /magazines)
- /books показує список книг так як зараз (з переходом на певну книгу)
- /magazines робить те саме, що /books - тільки для журналів (тобто у джеймон файлі в тебе буде два роути для отримання даних
/books та /magazines)
- якщо якийсь інший роут вбитий, воно редіректить на головну
- відповідно пошук має працювати на двох роутах - для книг та для журналів
- логіку отримання даних винеси в окрему папку (services я для прикладу)
- при загрузці сторінки з айді продукту воно має його підтягувати
